**Warm Boot Attack**

**TrueCrypt & VeraCrypt**
- cannot detect settings or application used
- in case of absence of memory image, identify application and settings used to encrypt volume
- used to create and mount and encrypted volume
- encrypt an entire partition or drive with a pre-boot authentication
- create and mount a 'hidden' volume within an encrypted volume
- volumes do not have any standard extention or headers, which makes them difficult to detect and identify
- VeraCrypt (PIM, keyfiles, Password)

**FULL DISK ENCRYPTION**
**Apple file system8**
- exFAT
  - cross-compatible file system between Apple macOS and Microsoft
  - Optimized for removable flash storage
  - not compatible with FileVault encryption
  - any encryption will be 3rd party
- HFS+
  - apple proprietary, successor of hierachical file system
  - introduced in macOS 8.1 and primary file system for mac until macOS High Sierra
  - optimized for HDD
  - compatible with FileVault Encryption
- APFS (Apple File System)
  - apple proprietary file system
  - introduced in macOS High Sierra
  - Optimized for SSD
  - Native support for FileVault encryption (same key for entire disk)
**Encryption Types**
- FileVault
  - introduced in macOS Panther
  - used to encrypt a user's home directory only
  - referred to as "legacy FileVault"
- FileVault2
  - introduced in macOS X Lion
  - Encrypts the entier partition(disc) protected with a password
  - uses AES encryption with 256 bit key
  - requires a user pw to unlock the disk
- Macs with T2 Chip
  - hardware security chip with a number of functions, including security
  - first appeared in iMac Pro 2017, MacBook Pro, Air, Mac Mini
  - Provides encryption services and secure boot
  - provides a key handling method to encrypt the data
  - hardware encryption, invisible to users

**Apple Keychain**
- pw management system in macOS and iOS
- introduced in macOS 8.6
- can be replicated across all mac 
- types
  - login
  - system
  - local items(icloud)
- can be shared among apple devices of the same icloud account
- can contain
  - website credentials, app credentials, credit card info, wifi info, filevault recovery keys, app certificates
**Decrypting filevault/apfs volume**
- encrypted volume image, disk partition, filevault wipekey file, decrypted image, destination file
- decrypted keychain, ios backup, icloud login
