Encrypted storage area
- disks(to make content unreadable by encrypting a storage area), partitions, containers
- encryption keys can be located in memory dumps

Supported types 
- APFS/FileVault2, Apple DMG disk, BitLocker, Dell Data Protection, DriveCrypt, LUKS and LUKS2, McAfee Endpoint, PGP, Symantec Endpoint Encryption, TrueCrypt, VeraCrypt
image file types
- raw data: byte-by-byte copy of image (DD, IMG, BIN)
- Truecrypt, VeraCrypt(no headers for volumes), DriveCrypt(TC, HC, DVC/DRC)
- E01, AFF4

Decrypting a bitlocker volume
- no memory image
  - rely on recovery key, bitlocker master key, brute force attack
- yes memory image

**How bitlocker work?**
- encryptes the entire volume
- when the correct decryption key is provided, the system will decrypt volume on the fly
- decrypt what it needs when it needs it
- Bitlocker Encrypted Volume + (Password decrypt --> volume master key decrypt --> Full volume encryption key) --> Decryption --> decrypted file

**Bitlocker Protectors**
- piece of info that bitlocker use to decrypt the vmk
  - password, trusted platform module (chip connected to motherboard and assist with security and integrity), TPM + PIN, TPM + startup key, TPM + startup key + PIN, startup key, other
