Encrypted storage area
- disks(to make content unreadable by encrypting a storage area), partitions, containers
- encryption keys can be located in memory dumps

Supported types 
- APFS/FileVault2, Apple DMG disk, BitLocker, Dell Data Protection, DriveCrypt, LUKS and LUKS2, McAfee Endpoint, PGP, Symantec Endpoint Encryption, TrueCrypt, VeraCrypt
image file types
- raw data: byte-by-byte copy of image (DD, IMG, BIN)
- Truecrypt, VeraCrypt(no headers for volumes), DriveCrypt(TC, HC, DVC/DRC)
- E01, AFF4

**Warm Boot Attack**
- acquire memory image without information of user credentials whether its off or on
![PDF Info Screenshot](/images/fulldisk3.png)
- process
  - prepare passware bootable memory imager usb
  - boot target machine normally, if its powerd off
  - perform a warm-boot using the hardware reboot/reset button and connect passware bootable memory imager usb shortly
  - intercept the boot and set BIOS to boot from USB
  - acquire a memory image using Passware Bootable Memory

**Recovery Keys**
- 48 digits used to decrypt a drive if the protectors are lost or forgotten
- bitlocker options to keep recovery key
  - write down, print, save to usb, upload to microsoft account, store in active directory

**TrueCrypt & VeraCrypt**
- cannot detect settings or application used
- in case of absence of memory iamge, identify application and settings used to encrypt volume
- used to create and mount and encrypted volume
- encrypt an entire partition or drive with a pre-boot authentication
- create and mount a 'hidden' volume within an encrypted volume
- volumes do not have any standard extention or headers, which makes them difficult to detect and identify
