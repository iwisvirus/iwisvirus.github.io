**Encryption**
1. advantages
  - computational algorithm
  - converts plaintext into ciphertext with a key
  - reversible with a key
  - variable length output (AES-256) 
2. Used for
  - Protect data "at rest" from the unauthorized access
  - Protect data "in transit" from the interception
  - Authentication and validation of the identity
  - Data erasure and fast deletion
3. Types
  - AES (advanced encryption standard) ex) AES-128 --> 128 bites


**Cryptographic Hash**
  - store only hash of the user password 
  - Computation algorithm
  - converst data into a 'hash', no key required
  - irreversible
  - fixed-length output (MD5) 

**Plaintext** : unencrypted content 

**Ciphertext** : Encrypted content 

**Cipher** : computer algorithm or process that converts Plaintext to Ciphertext 

**Key** : String of numbers/letters which are used by the cipher to convert plaintext to ciphertext 

**Symmetric Encryption** : method of the encryption where the same key is used for the encryption can also be used for the decryption

**Asymmetric Encryption** : method of the encryption where one key is used for the encryption, and a second different key is used for the decryption 

**Caesar Cipher** : key = offset of 3 meaning a --> d (decrypting -10, encrypting +10)

**Cryptanalysis**
  - attemmpting to convert ciphertext back into plaintext without knowledge of the password or the key
  - find/guess password/key, exploit a known weakness in cipher/implementation of the cipher

**Key derivation function(KDF)**

- irreversible
- deterministic, same input always generates the same output
- always produce an output of a fixed length

**Does not support Mac on**

- recovery of network and website pw on a local machine
- window account pw reset
- passware bootable memory imager (PBMI)

**Find the password!!**

  - written down, password manager, removal device, RAM/hibernation file
  - dictionary attack : works in particular language, all previous passwords, words extracted from data, tokenized words contained on storage device
  - bruteforce attack : all possible passwords
      - 10 character password --< 4 quintillion
  - exploite weakness in the cipher
      - decrypt data without a key
      - recover the key
