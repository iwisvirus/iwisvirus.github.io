**Mobile Device Backups**

- snapshot of mobile device, doesn't contain everything
- contents can be set by the user
- passware kit supports itunes and android backup files
- recovery speed v slow, hardware acceleration vital

**iTunes Backup File**

- found in windows and mac
- backup of an iphone, ipad, ipod
- collection of files and folders
- backups can be saved anywhere tbh
- windows: C:\Users\[user]\AppData\Roaming\AppleComputer\MobileSync\Backup
- mac: ~/Library/Applicaation Support/MobileSync/Backup

**Recover File password**

- only need Manifest.plist to entact encryption

**Android Backup File**

- snapshot of data from Android device
- single file with an .AB extension
- no default backup location

**Apple Keychain**

- present on both macOS and iOS devices
- replicated between other apple devices via iCloud
- keychain from one device can gain entry into a device with stronger encryption

**keychain types**

- ios keychain
  - saved within an itunes backup
- mac keychain
  - login keychain(Default): pw, secure notes, other data
      - by default, login keychain and user.kb pw are the same as the macOS user pw 
  - system keychain: stores items that are accessed by os, such as wifi pw
  - local items keychain: items that can be synced with icloud keychain: encrytion keys, applications data, webform entries, some ios data
- filevault2 recoveery key needed for decryption resides in the decrypted keychain file only if save the filevault2 recovery key to the cloud option was checked during the filevault2 intialization
