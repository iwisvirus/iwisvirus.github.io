**hash values and functions**
- digital fingerprint for files
- every file on a device has hash value
- determined by data contained in the file
- used to validate and verify data integrity
- mathetmatical function that transforms data of arbitary size into a hash value a string with a fixed lengt
- popular hash functions generate values bewteen 128 and 512 bits
- ex) MD5: 8743b52063 blabla
- each hash type has a unique algorithm that produces a different hash value for same file
    - SHA256 and SHA1 diff
 
**Cryptographic hash function**
- change of at least one bit in input data results into a significant hash value change
- irreversible
- designed to create unique hash values

**Rainbow tables**
- precomputed table for reversing cryptographic hash functions
- recover pw, the values from table are compared with the stored hashes to look for a match
- decryption process is almost instant and predetermined
-  available as add-on for office files and windows NTLM pw accordingly
- supported hash types
  - raw unsalted MD5, SHA1 : all types of word and excel 2003/2002/2000/97 files protected with the original 40-bit encryption
  - NTLM and LanMan: windows user pw
  - saves list of hases as a .txt file and attacks this file in passware kit
- drawbacks
  - can be applied to outdated hash algorithms only
  - require a large amount of storage
  - table can miss some hash values
  - ineffective against salted pw, but gpu/cpu-based brute-force attacks are still viable
